# Library UI/UX Revamp Plan

## Goal
Revamp Library screen so many saved phrases, vocabulary, and AI-curated items stay searchable, scannable, and actionable without becoming long messy text rows.

## Current Problems
- Single vertical feed mixes short vocabulary, long phrases, transcript saves, and AI-curated saves.
- Long text dominates row height and makes scanning hard.
- Phrase/Vocabulary tabs help classification, but not origin, review status, or context.
- AI-curated items have different behavior than video-saved items, but current UI only lightly signals that.
- Action menu hidden behind long press/ellipsis, so core actions are hard to discover.

## Recommended Direction
Use a card-based Library with quick filters and compact preview cards.

## Proposed UI Structure
1. Header
- Title: `Library`
- Subtitle/count summary: `12 phrases · 8 words · 5 AI-curated`
- Search input below header: `Search saved items...`

2. Filter Row
- Keep main segmented tabs: `Phrases` and `Vocabulary`.
- Add secondary chips: `All`, `From video`, `AI-curated`, `Recent`.
- Optional later chip: `Needs review`.

3. Saved Item Cards
Each item becomes compact card:
- Top row: badge + source
  - `Phrase` / `Vocabulary`
  - `Video` with play icon, or `AI-curated` with sparkles icon
- Main text preview:
  - Show 2-3 lines max.
  - For vocabulary, bold first word/title if detectable.
  - For phrases, show phrase as quote-style preview.
- Meta row:
  - Video item: timestamp + video title.
  - AI item: `AI-curated` + video title if available.
- Action row visible, not hidden:
  - Video item: `Open`, `Copy`, `Move`, `Delete` icons.
  - AI item: `Copy`, `Move`, `Delete` icons only.

4. Item Detail Modal
Tap card opens bottom sheet instead of navigating immediately.
- Full saved text.
- Source badge.
- For video item: `Open at timestamp` button.
- For AI item: `AI-curated` note and no video link.
- Actions: Copy, Move to other tab, Delete.

5. Search Behavior
- Search `text`, `videoTitle`, `videoChannel`, and source label.
- Empty state changes based on filters/search.

## Data/Code Changes
- No backend change needed.
- Reuse `SavedPhrase.source` added earlier.
- Add local component helpers in `app/app/(tabs)/library.tsx`:
  - `isAiCurated(item)` existing helper stays.
  - `getItemPreview(item)` for short display.
  - `matchesSearch(item, query)`.
  - `sourceLabel(item)`.
- Use `expo-clipboard` already installed for copy.
- Keep `formatTimestamp` only for video-saved items.

## Implementation Steps
1. Add Library local UI state:
- `searchQuery`
- `sourceFilter: 'all' | 'video' | 'ai'`
- `selectedItem: SavedPhrase | null`

2. Replace simple visible list filtering with:
- kind filter by active tab
- source filter
- search filter
- sort newest first

3. Replace row UI with card UI:
- More whitespace and badges.
- Clamp text with `numberOfLines`.
- Visible action buttons.
- Disable Open action for AI-curated items.

4. Replace current action modal content with detail-oriented modal:
- Full text display.
- Source/meta info.
- Contextual action buttons.

5. Add bottom toast for copy/delete feedback:
- Could reuse Home toast pattern or build tiny local toast.
- Avoid blocking Alert except destructive confirm if desired.

6. Validate:
- `npx tsc -p app/tsconfig.json --noEmit`
- Manual checks:
  - Video-saved phrase opens listen at timestamp.
  - AI-curated item has no Open video action.
  - Search finds text and video title.
  - Copy action copies full saved text.
  - Move between Phrase/Vocabulary updates tabs.

## Scope Choices
Recommended first pass:
- Search + source filters + compact cards + detail modal.
- Do not add spaced-repetition/review status yet.

Future improvements:
- Tagging.
- Bulk delete/move.
- Group by video.
- Practice mode from selected saved items.
- Swipe actions on native.
