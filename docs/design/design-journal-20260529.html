03 Penpot API lessons learned

Several plugin-API constraints were discovered through probing and worked around:

  • shape.horizontalSizing='fill' errors — must set child.layoutChild.horizontalSizing after appendChild.
  • Font weights can't be set via fontWeight='800' — use font.applyToText(t, variant) with the matching FontVariant.
  • Negative letterSpacing (e.g. -0.4) is rejected — only 0 / positive accepted; the hero tracking was dropped as a result.
  • lineHeight is a unitless multiplier — a raw '38' blew text up to 1216px tall.
  • Flex-layout boards recompute child x/y — manual position edits revert; spacing must be done via rowGap / child margins, and reserved widths.
  • Helpers must be inlined per call (stored function strings + eval failed).

07 Round 2 fixes — nav, play button, titles

Three issues reported off the rendered design, all fixed:

Fixed · Nav labels not aligned
Labels were left-offset from their icon centers. Re-centered each label under its icon — HOME cx 66≈65, LIBRARY 195=195, PROFILE 326≈325.
Removed · Play button on Today's Pick
Removed the green play button and the dark legibility overlay so the thumbnail shows clean, full-brightness.
Fixed · YouTube marker / title overlap
The ▶ marker text node measured only 1px wide (the glyph overflowed its own box), so the flex row packed the title on top of it. Gave the marker a fixed 10px reserved width so the flex gap:4 clears the title — consistent in Recent, Trending, and Today's Pick.

Upload worked directly from the YouTube CDN via penpot.uploadMediaUrl(...) this session (no base64 round-trip needed), then applied as image fills with keepAspectRatio:true:

// applied to each Thumbnail rectangle
rect.fills = [{
  fillOpacity: 1,
  fillImage: { id: img.id, width: img.width, height: img.height,
               mtype: img.mtype, keepAspectRatio: true }
}];
Known cosmetic effect: the 16:9 source cover-fits into the 160×160 square (Recent) and 150×200 portrait (Trending) slots, so the wide thumbnails show letterbox bars. Two options offered and pending a choice: (a) crop-fill edge-to-edge (zoom, no bars), or (b) make the thumbnail boxes 16:9.
