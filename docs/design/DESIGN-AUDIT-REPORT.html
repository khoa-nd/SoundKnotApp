🚫 Blockers · cannot ship −12 pts each

URL placeholder text fails contrast

“Paste a YouTube URL…” uses ink4 #686868 on paper2 #1A1A1A = 3.12:1. Normal 13px text needs ≥ 4.5:1.

FixUse ink3 #A0A0A0 (→ 4.6:1 on paper2) for the placeholder, or darken the field background.

LegalWCAG 2.1 SC 1.4.3 (Contrast Minimum)

LocationURL Input → “Paste a YouTube URL…” · source home.tsx:131 placeholderTextColor={colors.ink4}

⚠️ A placeholder is also not a label — see Cat 7 Critical below.

Bottom-nav inactive labels fail contrast

“LIBRARY” / “PROFILE” use ink4 #686868 on paper #121212 = 3.36:1 at 10px. Normal text needs ≥ 4.5:1.

FixUse ink3 #A0A0A0 (→ 7.16:1) for inactive tab labels + icons. Keep HOME on accent #1DB954 (7.24:1, passes).

LegalWCAG 2.1 SC 1.4.3

LocationBottom Nav → “LIBRARY”, “PROFILE” · source (tabs)/_layout.tsx:46 tabBarInactiveTintColor: colors.ink4

🔴 Critical Issues · must fix −8 pts each

Sub-44px touch targets — affects 3 element types

Submit button is 40×40px; bottom-nav items and chips are also tappable but small. Minimum tappable size is 44×44px (iOS HIG) / 48dp (Material).

FixSubmit → 44×44 (or keep 40px visual with a 44px hitSlop). Nav columns are wide horizontally (130px) and the 80px bar gives vertical room, so nav is OK — the 40px submit is the real fail.

WhyWCAG 2.5.5 / platform HIG — small targets cause mis-taps, worse for motor impairments.

LocationURL Input → Submit (40×40) · source home.tsx:264-273 width: 40

No accessible name on icon-only buttons

The Submit button (sparkles ✦) and the green Play button are icon-only with no text label or accessibilityLabel.

FixAdd accessibilityLabel="Process YouTube link" to the submit TouchableOpacity, and accessibilityLabel="Play" to the highlight card play control.

WhyWCAG 4.1.2 / 2.4.6 — screen readers announce nothing for these controls.

Locationsource home.tsx:139-153 (submit), HighlightCover.tsx:34-36 (play)

Placeholder used as the only label

The URL field has no visible label — only the placeholder “Paste a YouTube URL…”, which disappears on typing. Users lose context, and screen readers may not announce it.

FixAdd a visible label or at minimum accessibilityLabel="YouTube URL" on the TextInput. (The hero “Bring in a sound” is decorative, not a field label.)

WhyWCAG 1.3.1 / 3.3.2 — placeholders are not labels.

Locationsource home.tsx:130 placeholder="Paste a YouTube URL…"

🟡 Warnings · should fix −4 pts each

Play-button glyph fails icon contrast

White play icon #FFFFFF on rgba(29,185,84,0.9) green = 2.59:1. UI icons need ≥ 3:1. (Matches source — real bug.)

FixUse paper #121212 glyph on the green button (→ 7.24:1) — the same dark-on-accent pattern the submit button already uses.

Submit button reads as permanently disabled

The submit uses ink4 #686868 (the disabled-gray) in the design. In source it’s ink4 only when empty and ink when a URL is typed — but the default/empty state shown looks inert and low-contrast.

FixGive the empty state a subtler enabled affordance (e.g. paper2 fill + accent icon) so the primary action doesn’t look dead on first load.

Disabled / empty state of submit not visually distinct (Cat 11)

The empty-state submit (#686868) and a genuinely disabled control look identical — no way to tell “not yet” from “can’t.”

FixGive the empty state an accent-tinted outline; reserve flat gray for true disabled.

YouTube marker uses raw #FF0000, not a token (Cat 5 / 17)

The ▶ / YouTube marker is hardcoded pure red #FF0000 (5 cards + highlight). It’s not in the token set; pure red on dark is slightly harsh.

FixAccept it as the YouTube brand color (legitimate) or alias to a --brand-youtube token. Pure red on paper2 = 4.35:1 (OK for the small glyph, but tokenize it).

Marker ▶ size inconsistent (Cat 15)

The marker is 9px in cards but the highlight uses a 12px ▶ (and source uses a 14px logo-youtube Ionicon). Three sizes/representations for the same “YouTube” signal.

FixStandardize to one icon + one optical size (e.g. 12px) across all video rows.

Recent vs Trending card title height differs (Cat 5)

Recent titles are single-line (18px row); Trending titles are 2-line (34px row). Same content type, different treatment.

FixPick one line-clamp. The source uses numberOfLines={2} for both via the slider components — align the design to 2 lines everywhere.

Hairline borders may sub-pixel on the URL box & cards (Cat 9)

Card/field borders use hair rgba(255,255,255,0.12) at StyleSheet.hairlineWidth. On the dark surface this is fine — ensure elevation is read via the paper2 surface lift (it is) rather than the near-invisible border alone.

FixKeep the surface lift; treat borders as decorative-only. ✓ Mostly correct.

🟢 Tips · nice to have −1 pt each

Hero letter-spacing — Source titleLarge sets letterSpacing: -0.4; the Penpot text dropped it (negative tracking is rejected by the API). Minor — the hero reads slightly looser than the app.

Truncated edge labels (“The…”, “Chris…”, “The Tw… That S…”) — Intentional per the screenshot match, but they read as broken copy out of context. Fine for the off-screen peek card; ensure the real app uses a proper ellipsis component.

Section spacing rhythm — Hero→URL gap (24px) vs other section gaps (14px). Slightly irregular vertical rhythm. Optional: unify to a 16/24 pairing.

Content margins — Consistent 16px screen padding throughout. ✓ The bottom nav spans full-bleed (good) while content is inset 16px (good) — the contrast is intentional and correct.

✅ What’s Working Well
  • Strong type hierarchy — 32px hero → 22px section headings → 14/13px body → 10.5px mono chips. Clear, well-separated scale with meaningful weight contrast (the bold “sound”/“knot” emphasis is a nice touch).
  • Excellent primary-text contrast — Body/headings use ink #FFF (18.7:1) and ink2 #E0E0E0 (14.2:1) — far above AA.
  • Clean 8pt-grid spacing — Padding (16), gaps (8/14/24), card sizes (160, 150×200) all land on the grid. No arbitrary values.
  • Genuinely correct dark mode — Surfaces lift via paper2 #1A1A1A over paper #121212 (not pure black, no fake shadows), accent desaturated appropriately. Textbook dark-mode depth.

Nielsen heuristics: H6 (Recognition) ✓ nav has text labels; H2 (real-world match) ✓ YouTube marker + “Today’s Pick” language. H4/H5/H8/H9 covered by Cat 5/7/4/11. Main gap: H1 (system status) — no visible loading affordance in this static frame (the source does have skeletons + a progress modal, so this is design-frame-only).

Dark patterns (Cat 18): None found — Ethics 100/100. No confirmshaming, false urgency, decoy pricing, or pre-checked consent. Clean.
